    Отчет по лабораторной работе №1

    Реализация серверного приложения FastAPI (Тайм-менеджер)

    Цели работы

    Научиться реализовывать полноценное серверное приложение с помощью фреймворка FastAPI с применением дополнительных средств и библиотек.

    Выбор темы

    Разработка программы-тайм-менеджера — система для управления задачами, проектами и временем. Позволяет создавать задачи, распределять их по проектам и категориям, отслеживать затраченное время и получать уведомления о приближающихся дедлайнах.

    Модель данных

    В рамках работы спроектирована база данных, состоящая из 6 таблиц, что превышает минимальное требование (5 таблиц).

    Таблица Назначение
    users Пользователи системы (логин, email, пароль)
    projects Проекты пользователей
    tasks Задачи (принадлежат пользователю и опционально проекту)
    time_logs Записи затраченного времени на задачи
    categories Категории для группировки задач
    task_categories Ассоциативная сущность для связи many-to-many между задачами и категориями

    Связи между таблицами

    • one-to-many: users → projects, users → tasks, users → categories, projects → tasks
    • many-to-many: tasks ↔ categories (через таблицу task_categories)

    Дополнительное поле ассоциации

    В таблице task_categories присутствует поле weight (вес категории в задаче), которое характеризует связь помимо простых ссылок на связанные таблицы.

    Технологический стек

    • Python 3.10+
    • FastAPI — веб-фреймворк
    • SQLAlchemy — ORM
    • PostgreSQL — база данных
    • Alembic — миграции
    • JWT — аутентификация (ручная реализация)
    • bcrypt — хэширование паролей
    • Pydantic — валидация данных
    • Uvicorn — ASGI-сервер

    Реализованные эндпоинты

    Аутентификация (/auth)

    Метод Эндпоинт Описание
    POST /auth/register Регистрация пользователя
    POST /auth/login Вход, получение JWT-токена

    Пользователи (/users)

    Метод Эндпоинт Описание
    GET /users/me Информация о текущем пользователе
    GET /users/ Список всех пользователей
    PATCH /users/me/password Смена пароля

    Проекты (/projects)

    Метод Эндпоинт Описание
    POST /projects/ Создание проекта
    GET /projects/ Список всех проектов
    GET /projects/{id} Проект с вложенными задачами
    PATCH /projects/{id} Обновление проекта
    DELETE /projects/{id} Удаление проекта

    Категории (/categories)

    Метод Эндпоинт Описание
    POST /categories/ Создание категории
    GET /categories/ Список категорий
    PATCH /categories/{id} Обновление категории
    DELETE /categories/{id} Удаление категории

    Задачи (/tasks)

    Метод Эндпоинт Описание
    POST /tasks/ Создание задачи (с привязкой к категориям)
    GET /tasks/ Список задач (фильтрация по статусу и проекту)
    GET /tasks/{id} Получение задачи с вложенными категориями
    PATCH /tasks/{id} Обновление задачи
    DELETE /tasks/{id} Удаление задачи
    POST /tasks/{id}/time Добавление временной записи
    GET /tasks/report/time-summary Отчёт по времени на задачах
    GET /tasks/report/upcoming-deadlines Ближайшие дедлайны

    Примеры запросов и ответов

    Регистрация пользователя

    Запрос: 

    POST /auth/register
{
  "login": "alex1",
  "email": "alex1@example.com",
  "password": "1123456"
}

    Ответ (201 Created): 

    {
  "id": 1,
  "login": "alex1",
  "email": "alex1@example.com",
  "created_at": "2026-05-22T09:56:37.897037+03:00"
}

    Логин

    Запрос: 

    POST /auth/login
{
  "login": "alex1",
  "password": "1123456"
}

    Ответ (200 OK): 

    {
  "access_token": "eyJhbGciOiJIUzI1NiIs...",
  "token_type": "bearer"
}

    Создание задачи с категориями

    Запрос: 

    POST /tasks/
{
  "title": "Сделать ЛР1",
  "priority": 5,
  "project_id": 1,
  "category_ids": [1]
}

    Ответ (201 Created) с вложенными объектами: 

    {
  "id": 1,
  "title": "Сделать ЛР1",
  "status": "pending",
  "priority": 5,
  "categories": [
    {
      "category_id": 1,
      "category_name": "Срочно",
      "category_color": "#FF4444",
      "weight": 1,
      "assigned_at": "2026-05-22T10:03:28.425589+03:00"
    }
  ],
  "total_time_minutes": 0
}

    Отчёт по времени

    Запрос: GET /tasks/report/time-summary

    Ответ: 

    [
  {
    "task_id": 1,
    "task_title": "Сделать ЛР1",
    "total_minutes": 120
  }
]

    Аутентификация и безопасность

    • Хэширование паролей — реализовано с помощью библиотеки bcrypt (совместимость обеспечена фиксацией версии bcrypt==4.0.1)
    • JWT-токены — создаются и верифицируются вручную, без использования сторонних библиотек для самой аутентификации (кроме pyjwt для кодирования/декодирования)
    • Защита эндпоинтов — через зависимость get_current_user(), которая проверяет токен и извлекает пользователя
    • Конфиденциальность — секретные ключи и строки подключения хранятся в .env (исключён из Git через .gitignore)

    Миграции (Alembic)

    Настроена система миграций для управления схемой базы данных:

    alembic init migrations          # инициализация
alembic revision --autogenerate -m "initial_migration"  # создание миграции
alembic upgrade head              # применение миграции

    Выводы

    В ходе выполнения лабораторной работы №1 разработано полноценное серверное REST API для тайм-менеджера на FastAPI. Реализованы все обязательные требования на 15 баллов: спроектирована база данных из 6 таблиц с отношениями one-to-many и many-to-many (с дополнительным полем weight), созданы CRUD-эндпоинты для всех сущностей с возвратом вложенных объектов, настроены миграции Alembic, обеспечена правильная структура проекта. Дополнительно реализована полная система аутентификации с регистрацией, JWT-токенами и хэшированием паролей, а также функционал для работы с пользователями и отчёты по времени и дедлайнам.
